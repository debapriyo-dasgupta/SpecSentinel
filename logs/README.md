# SpecSentinel Logs Directory

This directory contains all application log files generated by SpecSentinel.

## Log Files

When the application runs, the following log files will be created here:

### Main Application Logs
- **`specsentinel.log`** - Main application log (all components)
- **`specsentinel.log.1`** to **`specsentinel.log.5`** - Rotated backup logs

### Component-Specific Logs
- **`specsentinel.api.log`** - Backend API logs (FastAPI)
- **`specsentinel.frontend.log`** - Frontend logs (Flask)
- **`specsentinel.pipeline.log`** - Analysis pipeline execution logs
- **`specsentinel.requests.log`** - HTTP request/response logs

### Agent Logs
- **`specsentinel.agent.SecurityAgent.log`** - Security agent logs
- **`specsentinel.agent.DesignAgent.log`** - Design agent logs
- **`specsentinel.agent.ErrorHandlingAgent.log`** - Error handling agent logs
- **`specsentinel.agent.DocumentationAgent.log`** - Documentation agent logs
- **`specsentinel.agent.GovernanceAgent.log`** - Governance agent logs

## Log Rotation

Logs automatically rotate when they reach **10MB** in size:
- Maximum file size: 10MB
- Backup count: 5 files
- Old logs are renamed with `.1`, `.2`, etc. suffixes
- Oldest logs are automatically deleted

## Configuration

Log behavior is controlled by environment variables:

```bash
# Log level (DEBUG, INFO, WARNING, ERROR, CRITICAL)
LOG_LEVEL=INFO

# Enable JSON logging (for production)
JSON_LOGGING=false

# Enable file logging
FILE_LOGGING=true
```

Set these in your `.env` file or as environment variables.

## Viewing Logs

### Real-time Monitoring (PowerShell)
```powershell
# Watch all logs
Get-Content logs\specsentinel.log -Wait

# Watch API logs
Get-Content logs\specsentinel.api.log -Wait

# Watch with filtering
Get-Content logs\specsentinel.log -Wait | Select-String "ERROR"
```

### Search Logs
```powershell
# Find errors
Select-String -Path "logs\*.log" -Pattern "ERROR"

# Find specific text
Select-String -Path "logs\*.log" -Pattern "analyze_spec"

# Count log levels
(Get-Content logs\specsentinel.log | Select-String "INFO").Count
(Get-Content logs\specsentinel.log | Select-String "ERROR").Count
```

### Parse JSON Logs
If JSON logging is enabled, you can parse logs with PowerShell:

```powershell
# Read and parse JSON logs
Get-Content logs\specsentinel.log | ForEach-Object { $_ | ConvertFrom-Json }

# Filter by level
Get-Content logs\specsentinel.log | ForEach-Object { 
    $log = $_ | ConvertFrom-Json
    if ($log.level -eq "ERROR") { $log }
}
```

## Troubleshooting

### No Logs Generated?

1. Check if `FILE_LOGGING` is enabled:
   ```bash
   echo $env:FILE_LOGGING  # Should be "true"
   ```

2. Check directory permissions:
   ```powershell
   Test-Path logs -PathType Container  # Should return True
   ```

3. Check if logger is initialized:
   ```python
   from src.utils.logging_config import get_logger
   logger = get_logger("test")
   logger.info("Test message")
   ```

### Logs Too Large?

Adjust rotation settings in `src/utils/logging_config.py`:
```python
MAX_BYTES = 10 * 1024 * 1024  # 10MB (change this)
BACKUP_COUNT = 5               # Number of backups (change this)
```

### Need More Detail?

Set log level to DEBUG:
```bash
$env:LOG_LEVEL="DEBUG"
```
